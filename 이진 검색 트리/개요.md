12 이진 검색 트리

검색 트리 자료구조는 SEARCH, MINIMUM, PREDECESSOR, SUCCESSOR, INSERT, DELETE 를 비롯해 많은 동적 집합 연산을 지원한다. 따라서 검색 트리를 사전(dictionary) 및 우선순위 큐(priority queue)로 사용할 수 있다. 

이진 검색 트리에 대한 기본 연산은 트리의 높이에 비례하는 시간에 수행된다. n개의 노드로 이루어진 완전 이진 검색 트리(complete binary search tree)에 대해 이런 연산들은 최악의 경우 ,O(logn)시간에 실행된다.

그러나 트리가 n개의 노드가 선형으로 연결된 모양일 경우 평균 O(n) 시간에 수행된다. 앞으로 임의로 만들어진 이진 검색 트리의 높이의 기댓값이 O(logn)임을 12.4절에서 확인할 것이다.

따라서 이런 트리에 대한 기본 동적 집합 연산들은 평균 O(logn) 시간에 수행된다. 

실제 상황에서 이진 검색 트리가 항상 임의로 만들어진다고 보장할 수는 없다. 하지만 기본 연산들에 대해서 최악의 경우에도 빠른 실행 속도를 보장할 수 있는 이진 검색 트리의 변형을 설계할 수 있다.
13장에서는 레드블랙 트리라고 하는 변종을 제시하는데, 높이가 O(logn)이다. 18장에서는 이차 저장공간(디스크)에서 데이터베이스를 관리하는데 특히 유용한 B-트리를 소개한다.

이진 검색 트리의 기본 특성을 제시한 후, 이후 절에서 값을 정렬된 순서로 출력하기 위해 이진 검색 트리를 순회(walk)하는 방법, 이진 검색 트리에서 하나의 값을 검색하는 방법, 최대 원소 또는 최소 원소를 찾는 방법, 한 원소의 직전원소 또는 직후원소를 찾는 방법, 이진 검색 트리에 삽입 또는 삭제를 수행하는 방법을 다룬다. 
트리의 기본적인 수학적 특성은 B를 참고하기 바란다.
